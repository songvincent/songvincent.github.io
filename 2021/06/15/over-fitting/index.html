# 过拟合以及相应的解决方法

# 定义

过于追求对训练数据的拟合程度，所学习到的模型复杂往往高于真实模型，这种现象就是过拟合。

# 缓解过拟合的方法

  • 数据角度

    • 从数据来源处获得更多数据
    • 进行数据增强

  • 模型角度

    • L1/L2 正则化 (后面详细阐述)
    • Dropout (后面详细阐述)
    • 结合多种模型 (Bagging/ Boosting)

# Dropout

# 定义

在每一次训练的迭代中，对每一层的中的神经元 (个数为 N) 以概率 P 随机剔除，用余下 (1-p)*N 的神经元所构成的网络来训练本次迭代中的数据。

# 优点

这可使得各个特征（权值）不用每一次迭代时都一起更新，互相产生关联和影响，也就弱化了各个特征之间由于数据量小导致产生过多的相互作用，缓解了过拟合。

# 缺点

每次迭代中只更新一部分参数，减慢了收敛速度。

# 原理

# L1/L2 正则

L1 正则化就是在损失函数后所加正则化项为 L1 范数，加上 L1 范数更容易得到稀疏解 (0 比较多)。L2 正则化就是损失函数后所加正则化项为 L2 范数，L2 正则化相比于 L1 正则化，得到的解比较平滑（不是稀疏），但是同样可以保证解中接近于 0 的维度比较多，降低模型复杂度。

# 一种通俗易懂的理解

  • L2 降低权重的平方，其导数为 2 * 权重
  • L1 降低权重绝对值，其导数为 k（与权重无关）

所以每次进行参数更新时，L2 每次移除权重的 x%, 即使进行多次计算权重也不会为 0；L1 每次移除一个固定的值，多次计算后权值归 0。
